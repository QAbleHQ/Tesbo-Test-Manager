# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Report suspected vulnerabilities by emailing:

vir@qable.io

You may also use GitHub private vulnerability reporting if it is enabled for this repository.

Include:

- A description of the issue
- Steps to reproduce
- Affected versions or commit hashes
- Any relevant logs or screenshots with secrets removed

QAble Testlab will acknowledge reports as soon as practical and coordinate remediation before public disclosure.

## Supported Versions

Until the project publishes formal releases, security fixes target the default branch.
